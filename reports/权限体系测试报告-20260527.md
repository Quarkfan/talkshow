# 三层权限体系 — 完整测试报告

> 日期: 2026-05-27 | 结果: **14/14 通过**

---

## 权限架构

| 层级 | 控制对象 | 控制字段 | 说明 |
|------|----------|----------|------|
| 第一层 | 主题卡片（主页显示） | `Theme.visible` | 控制是否在主页面展示卡片 |
| 第二层 | 主题内容（可访问性） | `Theme.accessible` | 控制该主题下属文件是否可被未登录用户访问 |
| 第三层 | 单个文件 | `ContentRule.public` | 对单个文件设置例外规则，覆盖第二层继承 |
| 分享层 | 分享链接独立控制 | `ShareLink.allow_content` | 分享链接是否允许访问第三层内容文件 |

### 优先级（从高到低）

1. **管理员** — 无条件访问所有文件
2. **文件级规则** (`ContentRule.public`) — 显式设置的文件例外
3. **分享 Cookie** — 有效分享链接 + allow_content=True → 放行所属主题文件
4. **主题继承** (`Theme.accessible`) — 文件无规则时继承主题可访问性
5. **兜底** — 存在任意 accessible 的主题 → 放行（针对无归属文件）

## API 变更

### 新增

```
POST /api/content-rules             创建文件规则
PATCH /api/content-rules/{id}       切换文件规则
POST /api/content-rules/bulk        批量切换所有文件规则
PATCH /api/shares/{share_id}/content 切换分享内容访问权限
POST /api/audit/cleanup             清理指定日期前的审计日志
```

### 修改

```
GET  /api/themes                    返回 visible + accessible
GET  /api/themes/public             返回 accessible 字段
GET  /api/shares                    返回 password + allow_content
GET  /api/audit                     返回 ip_address
```

### 新增页面

```
GET /blocked  → 主题不可访问时的禁止页面
GET /locked   → 分享不允许访问内容时的锁定页面
```

## 控制台 UI 变更

- **主题列表**：两列开关 — "主页显示"(visible) + "内容可访问"(accessible)
- **分享链接**：新增"内容访问"列开关，控制该分享是否允许访问第三层文件
- **内容权限**：展示每个文件的继承状态（来自哪个主题、是否被覆盖）
- **审计日志**：显示 IP 地址 + 日期选择器 + 清理按钮
